# AI智能问答模块使用手册

## 一、模块概述

智能问答模块是石化设备检修智能调度系统的AI助手功能，基于大语言模型提供专业的设备维护、调度安排、资源配置等方面的智能问答服务。

### 功能定位

- **专业知识问答**：解答化工设备维修相关的专业问题
- **调度辅助决策**：提供调度算法、工序安排、资源分配等方面的建议
- **数据查询支持**：结合系统数据（工人、设备、规则等）提供精准回答

### 2.2 左侧边栏 - 对话记录

| 功能 | 说明 | 操作方式 |
|------|------|----------|
| 新建对话 | 创建一个全新的对话会话 | 点击「新建对话」按钮 |
| 删除对话 | 删除当前选中的对话会话 | 选中会话后点击「删除对话」按钮 |
| 会话列表 | 显示所有历史对话记录 | 点击会话项切换到该会话 |
| 会话标题 | AI自动生成或手动识别的对话主题 | 首次问答后自动生成 |

### 2.3 右侧主区域

#### 2.3.1 消息展示区

- **用户消息**：绿色气泡，显示在右侧
- **AI消息**：白色气泡，显示在左侧，支持Markdown格式渲染
- **思考动画**：AI处理问题时显示"思考中..."动画
- **打字机效果**：AI回复时逐字显示，增强交互体验
- **时间标记**：消息间隔超过1分钟时自动显示时间分隔线

#### 2.3.2 输入区域

| 元素 | 说明 |
|------|------|
| 输入框 | 支持多行输入，最大500字符，显示字数统计 |
| 发送按钮 | 点击或按 Enter 键发送（Shift+Enter 换行） |
| 功能选择按钮 | 可选择是否包含特定上下文信息 |

## 三、功能详解

### 3.1 核心功能 - 智能问答

**使用方式**：在输入框中输入问题，点击发送按钮或按 Enter 键。

**支持的问题类型**：

| 问题类型 | 示例 |
|----------|------|
| 设备维护 | "反应釜的常见故障有哪些？" |
| 调度算法 | "拓扑排序和贪心算法有什么区别？" |
| 工序安排 | "更换新品管束需要哪些前置工序？" |
| 资源配置 | "如何合理分配工人到各个工序？" |
| 系统操作 | "如何创建一个新的检修计划？" |

### 3.2 功能选择按钮

在发送问题前，可以选择是否让AI参考特定的系统数据：

| 按钮 | 功能说明 | 状态 | 适用场景 |
|------|----------|------|----------|
| **规则库** | 将系统规则库信息作为上下文 | 🚧 开发中 | 需要基于规则进行问答时 |
| **调度方案** | 将当前调度方案信息作为上下文 | 🚧 开发中 | 查询调度方案相关问题时 |
| **工人信息** | 将系统中的工人信息作为上下文 | ✅ 已实现 | 查询工人分配、技能匹配等问题时 |
| **维修器具** | 将维修器具信息作为上下文 | 🚧 开发中 | 查询工具调配、使用等问题时 |

> **注意**：当前版本仅「工人信息」功能已实现，其他三个功能正在开发中，后续版本将陆续上线。

**使用建议**：
- 默认不勾选任何选项，AI基于通用知识回答
- 查询工人相关问题时，勾选「工人信息」以获得更精准的回答
- 可同时勾选多个选项（未实现功能不影响回答）

### 3.3 会话管理

#### 3.3.1 新建会话

点击「新建对话」按钮，创建一个独立的对话会话，与其他会话互不干扰。

#### 3.3.2 切换会话

点击左侧会话列表中的任一会话项，即可切换到该会话查看历史对话。

#### 3.3.3 删除会话

选中要删除的会话后，点击「删除对话」按钮确认删除。删除后自动切换到其他会话。

#### 3.3.4 会话标题生成

- **自动生成**：首次发送问题并收到AI回复后，系统自动调用AI生成会话标题
- **手动识别**：如果AI标题生成失败，使用首次用户提问的前24个字符作为标题

#### 3.3.5 本地存储

所有对话记录均保存在浏览器本地存储中，关闭页面后重新打开仍可恢复。

## 四、使用技巧

### 4.1 如何获得更好的回答

1. **问题描述要清晰**：尽量详细描述您的问题，包含上下文信息
2. **使用专业术语**：使用化工设备领域的专业术语，如"反应釜"、"管束"、"拓扑排序"等
3. **结合系统数据**：当查询系统相关信息时，勾选对应的功能按钮
4. **分步提问**：对于复杂问题，可以分步骤提问，逐步深入

### 4.2 常见操作快捷键

| 快捷键 | 功能 |
|--------|------|
| Enter | 发送消息 |
| Shift + Enter | 换行输入 |

### 4.3 注意事项

1. **网络要求**：使用前确保网络连接正常，AI服务依赖网络通信
2. **输入限制**：单次提问最多500字符，超出部分将被截断
3. **响应时间**：AI回答需要一定处理时间，请耐心等待
4. **数据安全**：对话记录仅保存在本地浏览器中，不会上传到服务器

## 五、FAQ

### Q1：AI回答不准确怎么办？

A：可以尝试以下方法：
- 更详细地描述问题
- 勾选相关的功能按钮，让AI参考系统数据
- 使用更专业的术语
- 尝试不同的提问方式

### Q2：对话记录会丢失吗？

A：不会。对话记录保存在浏览器本地存储中，除非手动清除浏览器数据，否则会一直保留。

### Q3：可以同时进行多个对话吗？

A：可以。点击「新建对话」创建多个会话，通过左侧列表切换。

### Q4：功能按钮什么时候需要勾选？

A：当您的问题涉及系统中的具体数据时勾选，例如：
- 查询某个工人的信息时勾选「工人信息」
- 查询调度方案细节时勾选「调度方案」

### Q5：AI支持哪些语言？

A：目前主要支持中文，建议使用中文提问以获得最佳效果。

## 六、技术说明

### 6.1 系统架构

```
┌──────────────┐      HTTP请求      ┌──────────────┐      API调用      ┌──────────────┐
│   前端界面   │ ──────────────────> │   后端API    │ ─────────────────> │   AI模型    │
│  (Vue3)      │ <───────────────── │  (Flask)     │ <───────────────── │ (DashScope) │
└──────────────┘      JSON响应       └──────────────┘      生成回答       └──────────────┘
       │                                                     │
       │ 本地存储 (localStorage)                             │
       └─────────────────────────────────────────────────────┘
```

### 6.2 API接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/chat` | POST | 发送问题，获取AI回答 |
| `/api/chat/title` | POST | 根据对话内容生成标题 |

### 6.3 数据上下文

AI在回答时可参考以下系统数据：

- **工人信息**：从workers表读取，包含工人工种、技能等级、认证状态等
- **设备信息**：从equipment表读取，包含设备类型、状态等
- **规则库**：系统规则配置信息
- **调度方案**：当前调度计划信息
- **维修器具**：维修工具和设备信息
